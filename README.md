# Lasker

A Chrome extension that adds a live Stockfish analysis overlay to **chess.com**.

When enabled, Lasker reads the current board from the page, runs Stockfish
locally in your browser (WebAssembly, no external servers), and shows:

- A numeric evaluation (e.g. `+1.35`, `-0.80`, `M3`) from white's perspective
- A vertical evaluation bar (white vs black advantage)
- The top 3 engine lines with the first moves of each
- The current search depth, with a slider to adjust it (8 to 22)

## Intended use

This is an **analysis and learning tool**. It's meant for:

- Reviewing finished games
- Studying puzzles
- Exploring positions in chess.com's analysis board
- Training and preparation

Do **not** use it during live rated or competitive games. Doing so violates
chess.com's Fair Play policy and can get your account closed.

## Install (unpacked)

1. Install dependencies to grab the Stockfish WASM binaries (once):
   ```bash
   npm install
   ```
   This populates `engine/stockfish.js` and `engine/stockfish.wasm` only if
   you rebuild from scratch -- they're normally committed to the repo.

2. Open Chrome and go to `chrome://extensions`.

3. Enable **Developer mode** (top right).

4. Click **Load unpacked** and select this folder (the one containing
   `manifest.json`).

5. Visit any chess.com page with a board (e.g. `/play`, `/puzzles`,
   `/analysis`, a game URL).

6. A small panel appears in the bottom-right. Click the toggle to start
   analysis. Click and drag the header to move it.

## Architecture

- **`manifest.json`** -- Manifest V3, targets `https://www.chess.com/*`.
- **`content/board-reader.js`** -- Reads chess.com's DOM (`<wc-chess-board>` /
  `<chess-board>`) and builds a FEN from `.piece.square-FR` elements.
- **`engine/stockfish.js` + `.wasm`** -- Bundled Stockfish 18 Lite
  single-threaded WASM (~7 MB). No SharedArrayBuffer required.
- **`engine/stockfish-worker.js`** -- Thin wrapper that spawns the Stockfish
  Web Worker, manages the UCI handshake, and parses `info` lines.
- **`content/overlay.js`** -- Shadow-DOM floating panel with score, eval bar,
  top-3 lines, and depth slider.
- **`content/content.js`** -- Controller: polls the board every 500 ms,
  detects position changes, flips side-to-move, and drives the engine.
- **`background.js`** -- Minimal service worker; clicking the toolbar icon
  toggles analysis in the active chess.com tab.

## Known limitations

- **Castling rights** and **en passant target** are inferred best-effort from
  piece positions. For most positions this is fine; in edge cases the
  evaluation may be off by a tenth of a pawn.
- **Side-to-move** is tracked heuristically (flipped on every detected board
  change). Toggling the extension off and on re-seeds from piece counts.
- **MutationObserver is unreliable on chess.com** for rapid updates, so
  Lasker polls the DOM every 500 ms instead.
- Chess.com's DOM can change without notice. All selectors are isolated in
  `content/board-reader.js` for easy maintenance.

## Development

- No build step. Edit files in place and click the reload button on the
  extension in `chrome://extensions` to pick up changes.
- Icons are generated by `icons/make_icons.py` (plain stdlib) if you want
  to rebuild them.
- Stockfish binaries come from the `stockfish` npm package
  (Lite single-threaded variant).
